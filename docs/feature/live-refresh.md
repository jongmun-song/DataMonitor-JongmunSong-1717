# 실시간 갱신

## 개요

이 도구의 본질은 "정적인 1회성 출력"이 아니라 실시간(계속 갱신되는) 조회다. 이 문서는
`sample-view.md`, `order-list-view.md`, `monitoring-summary.md`,
`production-line-view.md`, `release-candidates-view.md`에 공통으로 적용되는 갱신
규칙과, 메인 메뉴의 요약 현황 표시를 정의한다.

## 동작

### 캐시 없는 재조회

- 각 조회 화면에 진입하거나 갱신 명령을 입력할 때마다, 해당 화면이 사용하는
  데이터소스(`data-source.md`)의 `reload()`를 호출해 JSON 파일을 다시 읽는다.
- 이전 조회 결과를 필드 등에 캐싱해 재사용하지 않는다. 저장된 데이터가 외부에서
  변경되어도(더미 JSON 파일을 실행 중 다른 도구/편집기로 수정하는 경우 등) 다음
  갱신 시점에 최신 상태가 반영되어야 한다.

### 갱신 방식

- 각 조회 화면에서 사용자가 특정 키(예: `r`)를 입력하면 즉시 재조회 후 다시
  렌더링하는 **수동 갱신**을 기본으로 지원한다.
- 자동 주기 갱신(예: N초마다 자동 재조회)은 선택 구현 사항으로 남겨둔다. 구현할
  경우 사용자가 자동 갱신을 켜고 끌 수 있어야 한다.

### 조회 시각 표기

- 모든 조회 화면은 이번 조회가 이루어진 시각(예: `2026-04-16 09:32:15`)을 함께
  표시한다(요구사항 p.19 모니터링 예시 UI 참고).

### 메인 메뉴 요약 현황

- 메인 메뉴에는 전체 시료 종수, 총 재고 수량, 전체 주문 건수 등 시스템 전체 요약
  정보를 표시한다(요구사항 p.11 메인 메뉴 예시 UI 참고).
- 메인 메뉴 진입 시에도 위 "캐시 없는 재조회" 규칙을 동일하게 적용한다.

## 관련 기능

- `data-source.md`: 이 문서가 규정하는 재조회 대상인 `reload()`가 정의된 곳.
- `sample-view.md`, `order-list-view.md`, `monitoring-summary.md`,
  `production-line-view.md`, `release-candidates-view.md`: 이 갱신 규칙을 적용받는
  화면들.
