# 모니터링 요약 (requirements.pdf p.19)

## 개요

상태별 주문 건수와 시료별 재고 현황을 한눈에 요약해서 보여주는 화면. 요구사항
문서의 "모니터링"(p.19)에서 정의하는 "주문량 확인"/"재고량 확인" 두 하위 화면에
대응한다.

## 동작

### 주문량 확인

- `order-list-view.md`와 동일한 `OrderDataSource` 조회 결과를 상태별로
  그룹핑하여 RESERVED/CONFIRMED/PRODUCING/RELEASE 각각의 건수를 표시한다.
- REJECTED는 유효한 주문이 아니므로 집계에서 제외한다(요구사항 p.8, p.18).

### 재고량 확인

- `sample-view.md`와 동일한 `SampleDataSource` 조회 결과를 이용해 시료별 현재 재고
  수량을 표시한다.
- 재고 수량에 따라 다음 상태를 함께 표기한다(요구사항 p.18):
  - **여유**: 재고가 충분한 상태
  - **부족**: 재고가 부족한 상태
  - **고갈**: 재고 수량이 0인 상태
- 잔여율(재고 수량 기반 백분율)을 함께 계산해 표시한다(p.19 예시 UI의 진행률 바
  참고).
- 여유/부족/고갈의 경계가 되는 구체적인 임계치와 잔여율 계산식은
  `docs/design/phase3-monitoring-summary.md`에 정의한다(요구사항 문서가 정확한
  수치를 규정하지 않으므로, 이 저장소가 자체적으로 정의한 기준이다).

## 관련 기능

- `data-source.md`, `order-list-view.md`, `sample-view.md`: 이 화면이 재사용하는
  조회/필터링 로직의 출처.
- `live-refresh.md`: 매 조회 시점의 데이터 재조회 및 조회 시각 표기.
